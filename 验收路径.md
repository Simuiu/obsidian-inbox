## 8. 微信视频号半自动入库验收

视频号验收分两层：一层验证本地导入链路，另一层在用户自己的微信客户端环境下验证捕获流程。

### 8.1 本地导入链路验收

如果已经有一个本地 `.mp4` 和可用转写，执行：

```bash
python3 scripts/import_wechat_video.py \
  --video "/path/to/video.mp4" \
  --source-url "视频号原链接或来源说明" \
  --title "视频标题" \
  --author "作者" \
  --transcript "/path/to/transcript.md"
```

通过标准：

- `platform` 为 `wechat_video`
- `status` 为 `transcript_captured`
- 任务包包含 `meta.json`、`compact.md`、`prompt.md`
- 本地视频原文件不复制进 Obsidian；转写进入单条资源文件夹的 `_resources/`
- 有转写时可以基于转写生成笔记

如果没有转写但要验证 ASR，执行：

```bash
python3 scripts/import_media.py \
  --platform wechat_video \
  --video "/path/to/video.mp4" \
  --source-url "视频号原链接或来源说明" \
  --title "视频标题" \
  --author "作者" \
  --asr
```

通过标准：

- ASR 可用且质量足够时，状态进入可总结链路
- ASR 不可用或转写质量不足时，状态为 `needs_asr` 或 `needs_asr_review`
- 没有可靠转写时不生成假总结

### 8.2 本地捕获流程人工验收

执行：

```bash
python3 scripts/install_resd_mini.py
python3 scripts/start_resd_mini.py
```

人工操作：

1. 在捕获工具界面启动监听或代理。
2. 在微信客户端里打开并播放目标视频号内容。
3. 确认捕获工具生成本地 `.mp4` 文件。
4. 将该 `.mp4` 交给 `import_wechat_video.py` 或 `import_media.py --platform wechat_video` 导入。

通过标准：

- Codex 能协助安装或启动捕获工具
- 用户动作边界清楚：必须在微信客户端打开并播放目标视频
- 捕获到 `.mp4` 后能进入本地导入链路
- 项目不承诺只靠一个视频号 URL 直接下载
- 未获得视频文件、转写或有效 ASR 时，不生成完整总结

```bash
sed -n '1,80p' /tmp/obsidian-inbox-render-test.md

```

通过标准：

- 生成 Markdown 文件。

- frontmatter 包含 `summary_mode`。

## 11. 人工验收清单

- 文件命名是否为 `YYYY-MM-DD 短标题.md`
- source ID 是否没有塞进正文文件名
- 轻量原始材料是否进入单条资源文件夹的 `_resources/`
- 微信文章图片是否进入单条资源文件夹的 `_resources/images/`
- 微信视频号是否写成“本地捕获后半自动入库”，而不是“只给链接直接下载”
- 微信视频号捕获流程是否明确需要用户在微信客户端打开并播放视频
- 失败任务是否没有生成假总结
- 测试任务是否默认清理，不污染 Obsidian
- `用户手册.md` 是否能指导新用户完成入库
- `PRIVACY_AND_LIMITS.md` 是否清楚说明平台边界
