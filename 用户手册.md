# Obsidian Inbox 用户手册

## 这个工具解决什么问题

`Obsidian Inbox` 用来把外部视频、文章、图文和本地材料整理进 Obsidian 资料库。它的核心目标不是下载一堆原始文件，而是形成可回看、可索引、可实践的 Markdown 笔记。

当前主流程：

```text
链接或本地文件 -> 任务包 -> compact 摘要上下文 -> Obsidian 草稿/笔记 -> 自动索引
```

## 推荐使用方式

在 Codex 里直接说：

```text
入库这个链接：https://...
```

或者：

```text
深度入库这个视频：https://...
```

Codex 会优先使用 `.agents/skills/obsidian-inbox/SKILL.md` 里的流程。

## Bilibili 视频

适合：课程、技术讲解、访谈、长视频。

命令：

```bash
python3 scripts/prepare_link.py "https://www.bilibili.com/video/BVxxxx" --sync-raw
```

深度模式：

```bash
python3 scripts/prepare_link.py "https://www.bilibili.com/video/BVxxxx" --sync-raw --mode deep
```

总结模式说明：

- 普通总结：用于快速入库和第一遍理解。
- 深度总结：用于课程、教学、系统整理、本地视频，或你明确要求“深度入库 / 深度总结”的内容。
- 主笔记会在 `索引与主题` 中显示 `总结模式：普通` 或 `总结模式：深度`。
- 索引页也会展示每条资源的总结模式。

说明：

- 优先抓 B 站 AI 字幕。
- 优先使用 `secrets/bilibili.cookies.txt`。
- 如果字幕抓不到，不生成假总结。

## YouTube 视频

命令：

```bash
python3 scripts/prepare_link.py "https://www.youtube.com/watch?v=xxxx" --sync-raw
```

说明：

- 当前只处理公开可访问的元数据和字幕。
- 如果 YouTube 要求登录、机器人校验或没有字幕，会标记失败或 `needs_subtitle_or_asr`。
- 不绕过登录或校验。

## 微信公众号文章

命令：

```bash
python3 scripts/prepare_link.py "https://mp.weixin.qq.com/s/xxxx" --sync-raw
```

能力：

- 抓标题、公众号、发布时间。
- 抓正文 Markdown。
- 图片会本地化到 `_resources/images/`。
- 诊断写入 `article_diagnostics.json`。

## 微信视频号

视频号不能稳定靠 URL 直接下载。推荐先用本地工具捕获你有权访问的视频文件，再导入。

推荐工具：

```bash
python3 scripts/install_resd_mini.py
python3 scripts/start_resd_mini.py
```

导入本地视频：

```bash
python3 scripts/import_media.py \
  --platform wechat_video \
  --video "/path/to/video.mp4" \
  --source-url "视频号原链接或来源说明" \
  --title "视频标题" \
  --author "作者"
```

已有转写：

```bash
python3 scripts/import_media.py \
  --platform wechat_video \
  --video "/path/to/video.mp4" \
  --source-url "视频号原链接或来源说明" \
  --title "视频标题" \
  --author "作者" \
  --transcript "/path/to/transcript.md"
```

自动 ASR：

```bash
python3 scripts/import_media.py \
  --platform wechat_video \
  --video "/path/to/video.mp4" \
  --source-url "视频号原链接或来源说明" \
  --title "视频标题" \
  --author "作者" \
  --asr
```

## 小红书 / 抖音图文

当前推荐合规路径：先把你有权访问的图文内容导出到本地，再导入。

本地图文导入：

```bash
python3 scripts/import_article_export.py \
  --platform xiaohongshu \
  --content "/path/to/content.md" \
  --image-dir "/path/to/images" \
  --source-url "原始链接或来源说明" \
  --title "标题" \
  --author "作者"
```

抖音图文同理：

```bash
python3 scripts/import_article_export.py \
  --platform douyin \
  --content "/path/to/content.md" \
  --image-dir "/path/to/images" \
  --source-url "原始链接或来源说明" \
  --title "标题" \
  --author "作者"
```

## 小红书 / 抖音视频

当前推荐合规路径：先把你有权保存的视频放到本地，再导入。

```bash
python3 scripts/import_media.py \
  --platform douyin \
  --video "/path/to/video.mp4" \
  --source-url "原始链接或来源说明" \
  --title "标题" \
  --author "作者" \
  --transcript "/path/to/transcript.md"
```

没有转写时可以加 `--asr`，但长视频建议先人工确认转写质量。

## 本地公开视频 / 自有视频

本地视频默认按深度总结作为主笔记；普通总结如需保留，会归档到该资源文件夹的 `_resources/普通总结归档.md`。

```bash
python3 scripts/import_media.py \
  --platform local_video \
  --video "/path/to/video.mp4" \
  --source-url "来源说明" \
  --title "标题" \
  --author "作者" \
  --transcript "/path/to/transcript.md"
```

## 生成 Obsidian 草稿

任务包生成后，可以先生成可编辑草稿：

```bash
python3 scripts/render_note.py "tasks/YYYY-MM-DD-platform-id" --write
```

深度草稿：

```bash
python3 scripts/render_note.py "tasks/YYYY-MM-DD-platform-id" --mode deep --write
```

默认不会覆盖已有笔记。确认覆盖时加：

```bash
--force
```

## 平台边界

详细边界见：

```text
PRIVACY_AND_LIMITS.md
```

原则：

- 不绕过登录、验证码、机器人校验或付费墙。
- 不抓取用户无权访问的内容。
- 抓取失败时只记录失败状态，不生成假总结。
- 不提交真实 cookies、私有任务包或 Obsidian 原始材料。
