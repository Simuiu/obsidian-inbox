# 10 Obsidian Inbox 首轮发布顺序执行包

## 1. 执行原则

- 先发低成本、反馈快的平台。
- 每个平台发布时至少使用 5 张配图，小红书使用 6 张。
- 每条内容都导向 GitHub：`https://github.com/Simuiu/obsidian-inbox`
- 先验证用户是否理解“不是摘要，是入库”，再扩大制作视频和图文。

## 3. 每个平台发布前检查

- 标题是否明确。
- 是否至少准备 5 张配图。
- 是否包含 GitHub 链接或 GitHub 搜索关键词。
- 是否没有“绕过限制”“全平台一键下载”等高风险表达。
- 是否有评论区回复预案。
- 发布后是否记录到 `09 发布记录与复盘模板`。

## 4. 今天可以先做的三件事

1. 发即刻动态。
2. 发 V2EX 求反馈帖。
3. 选择一个知乎问题，发布回答。

这三步不需要视频素材，适合作为第一轮反馈测试。

## 5. Codex 自评结论

这个顺序优先低成本验证，再进入长文和视频制作。它能最大限度减少无效制作，也能尽快看出用户是否理解项目定位。
