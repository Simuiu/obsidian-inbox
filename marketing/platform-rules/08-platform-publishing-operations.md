# 08 Obsidian Inbox 平台发布操作辅助

## 1. 使用说明

本文件用于把 `05 全平台首轮发布物料包` 和图片资产落到各平台发布操作中。

当前发布助手已经改成“全平台单稿分发”：每个平台只有一版确定稿，图片/视频统一按下载后上传处理，不再依赖粘贴图片。

安全边界：

- 不在对话中展示 token、cookie、session、auth 文件。
- 未确认授权前，不直接登录或代发内容。
- 默认只做到草稿、待发布或可检查状态。
- 高风险平台不做 cookie 自动发帖。

## 2. 平台操作清单

| 平台 | 当前我能做什么 | 发布方式 | 你需要做 |
| --- | --- | --- | --- |
| GitHub | 可以直接改 README、Topics、Release 草稿、Issue 模板，并提交推送 | 自动/半自动 | 最终确认是否公开发布 Release |
| 飞书 | 可以直接同步 Markdown 文档 | 自动 | 查看飞书结果 |
| 微信公众号 | 如提供公众号 AppID/Secret 且有草稿箱接口权限，我可以尝试创建草稿；否则只准备发布包 | 半自动优先 | 后台上传封面、配图，最终点发布 |
| 知乎 | 没有稳定官方发文接口；可准备内容和素材 | 半自动 | 粘贴回答/文章，上传配图，最终发布 |
| 掘金 | 未确认稳定官方草稿接口；可准备 Markdown 和素材 | 半自动 | 粘贴 Markdown，上传配图，检查代码块 |
| V2EX | 官方 API 主要偏读取，不建议自动发帖 | 人工确认发布 | 选节点，粘贴正文，必要时不放图 |
| 即刻 | 未接入稳定发布接口 | 人工确认发布 | 粘贴动态，上传配图 |
| B 站 | 未接入稳定上传工具；可准备视频、封面、标题、简介、标签、置顶评论 | 半自动 | 上传视频和封面，复制发布字段 |
| 小红书 | 不建议 cookie 自动发帖；可准备图文卡片和正文 | 半自动 | 上传 6 张图，复制标题/正文/话题 |
| 视频号 | 未接入稳定视频号发布接口；可准备视频、封面、标题和文案 | 半自动 | 上传视频，复制标题/文案 |

## 3. HTML 助手使用流程

1. 打开本地发布助手：

```text
http://127.0.0.1:8765/index.html
```

2. 切换到对应平台。
3. 复制标题、摘要/简介、正文。
4. 下载图片或视频，再到平台后台上传。
5. 发布前检查 GitHub 链接和风险表达。

## 4. 微信公众号发布流程

1. 切换到“微信公众号”。
2. 下载封面图：

```text
marketing/publish-packets/2026-05-11-full-platform-launch/images/wechat/cover-wechat-flow.png
```

3. 复制标题、摘要、正文。
4. 上传 5 张正文配图：

```text
marketing/publish-packets/2026-05-11-full-platform-launch/images/wechat/cards/
```

5. 发布前检查：
   - 是否保留 GitHub 链接。
   - 是否没有“绕过限制”等高风险表达。
   - 图片是否显示正常。

## 5. 小红书发布流程

1. 上传图片：

```text
marketing/publish-packets/2026-05-11-full-platform-launch/images/xiaohongshu/final-cards/
```

2. 复制标题、正文、话题。
3. 检查首图是否足够清晰。
4. 如果平台压缩导致字太小，优先重新生成小红书卡片。

## 6. B 站发布流程

1. 上传视频：

```text
marketing/publish-packets/2026-05-11-full-platform-launch/video-assets/bilibili-slideshow.mp4
```

2. 上传封面：

```text
marketing/publish-packets/2026-05-11-full-platform-launch/images/bilibili/final-covers/bilibili-cover-01.png
```

3. 复制标题、简介、标签、置顶评论。

## 7. 我可以帮你发布到什么程度

| 平台 | 我能直接做 | 需要你配合 |
| --- | --- | --- |
| GitHub | 可以直接更新仓库、Release 草稿、Issue 模板、Topics 检查、提交推送 | 是否确认发布 Release |
| 飞书 | 可以直接同步所有 Markdown 文档 | 无 |
| 微信公众号 | 有开发者凭据和草稿权限时，可尝试创建草稿 | 提供安全授权方式，最终发布仍由你确认 |
| 其他内容平台 | 当前主要是生成发布包、素材、检查清单 | 你登录平台并最终发布 |

## 8. 自动化待验证

| 平台 | 待验证点 |
| --- | --- |
| 微信公众号 | 是否有可用公众号开发者权限，能否通过草稿箱 API 创建草稿 |
| 掘金 | 是否有稳定草稿 API/CLI 或可接受浏览器辅助 |
| B 站 | 是否有可靠上传工具，是否支持待发布草稿 |
| 小红书 | 不建议 cookie 自动发帖，最多做低频浏览器辅助 |
| 视频号 | 默认人工发布，后续再看是否能走微信生态接口 |

## 9. Codex 自评结论

本文件已经能支持你按平台逐个发布。当前最省事的路线是：先用 HTML 助手复制文案，再手动上传对应平台的配图/视频。

自动化优先从 GitHub、飞书和微信公众号草稿箱接口开始验证，其他平台暂不做高风险自动登录或 cookie 自动发帖。
